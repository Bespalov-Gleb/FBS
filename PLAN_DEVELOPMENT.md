## 1. Синхронизация и удаление отменённых заказов

**Проблема:** FBS Tools подгружает заказы из «готовы к отгрузке» и «ожидают сборки», но отменённые заказы не удаляет. Упаковщик видит товар, упаковывает, а на WB его уже нет.

### Задачи

- [x] **1.1** Удалять/архивировать заказы при синхронизации — если заказ есть в БД, но его нет в ответе API маркетплейса (отменён, исчез), помечать как удалённый или удалять из списка сборки
- [x] **1.2** Настроить Celery Beat — добавить `beat_schedule` в `celery_app.py`, запуск `sync_all_marketplaces` каждые 5–10 минут
- [ ] **1.3** (Опционально) Добавить статус `cancelled` в Order, если API поддерживает отмену явно

**Файлы:** `backend/app/services/order_sync_service.py`, `backend/app/core/celery_app.py`, `backend/app/tasks/sync_tasks.py`

## 2. Только «Ожидают отгрузки» (Ozon) и «На сборке» (WB)

**Требование:** Ozon — только «Ожидают отгрузки», WB — только «На сборке». Не смешивать с «готовы к отгрузке».

### Статус: ✅ Сделано

- Ozon: `get_orders_awaiting_deliver()` — только awaiting_deliver
- WB: new + confirm — только «на сборке»

## 3. Фильтры на экране сборки

**Требование:** Показывать заказы:
1. **Все магазины, все склады** — единый список
2. **Каждый магазин (склад) отдельно**
3. **Ozon все магазины** / **WB все магазины** — Ozon упаковывают все вместе, WB отдельно

### Задачи

- [x] **3.1** Добавить в OrderFilters варианты:
  - «Все» (все маркетплейсы и склады)
  - «Ozon (все)» — все Ozon-магазины
  - «WB (все)» — все WB-магазины
  - Каждый маркетплейс по отдельности
  - Каждый склад по отдельности
- [x] **3.2** API: поддержать фильтр по типу маркетплейса (`marketplace_type=ozon|wildberries`) и по нескольким `marketplace_id`
- [x] **3.3** При выборе «Все» — загружать склады со всех маркетплейсов (сейчас warehouses требуют выбранный marketplace_id)
- [x] **3.4** Добавить endpoint `GET /warehouses` — список всех складов пользователя (для фильтра «по складу»)

**Файлы:** `frontend/src/components/orders/OrderFilters.tsx`, `frontend/src/pages/AssemblyPage.tsx`, `backend/app/api/v1/endpoints/orders.py`, `backend/app/api/v1/endpoints/warehouses.py`, `backend/app/repositories/order_repository.py`
