# Отчёт: КИЗ (маркировка) — что нужно поправить

## 1. Структура КИЗ

### Полный код (при сканировании)
Пример: `0104679623821024215aOARYVmjSE>&91EE1192ttWGmRDD+Pg9Vq2m/qZCcyZZ38GYjKROKECVZlKhm+k=`

**Состав кода маркировки Честный ЗНАК:**

| Тег | Назначение | Пример |
|-----|------------|--------|
| **01** | GTIN (код товара) | 14 цифр |
| **21** | Серийный номер | 13 символов (буквы, цифры) |
| **91** | Ключ проверки | 4 символа |
| **92** | Код проверки (криптохвост) | 4–88 символов (для одежды — 88) |

**Специальные символы:**
- `>` — разделитель групп (GS, ASCII 29) — не печатается, используется при формировании DataMatrix
- `&` и др. — часть криптохвоста, генерируется системой ЦРПТ

**Для одежды:** полный код ~88 символов в криптохвосте. Итого: 2+14+2+13+4+2+88 ≈ 125 символов в DataMatrix.

### 31 символ (для WB/Ozon)
Первые **31 символ** — открытая часть: GTIN (14) + серийный номер (13) + ключ проверки (4).

Пример: `0104679623821024215aOARYVmjSE>&`

**WB и Ozon принимают только эти 31 символ** при вводе/загрузке. Криптохвост не вводится — он уже внутри DataMatrix.

## 2. Дубль этикетки КИЗ (DataMatrix + 31 символ)

### Текущее состояние (обновлено 2025-03)
- Endpoint `/orders/kiz-label` возвращает **DataMatrix** (полный КИЗ) + 31 символ текстом снизу
- Формат: PDF (fallback на QR в PNG при отсутствии pylibdmtx)
- Зависимость: `pylibdmtx` для reportlab ECC200DataMatrix

### Было (до исправления)
- QR-код с 31 символом, формат PNG

### Что реализовано
- **DataMatrix** (2D-код) — кодирует **весь** КИЗ (все 85+ символов), включая криптохвост
- Под DataMatrix — **31 символ** текстом (для ручного ввода при необходимости)

### Важно про DataMatrix
- DataMatrix должен содержать **полный** код с криптохвостом
- Формат: **GS1 DataMatrix** — требуется префикс FNC1 и разделитель GS между блоками
- Генерация вне системы Честный ЗНАК возможна, но нужны правильные управляющие символы (FNC1, GS)
- Рекомендуется: библиотека `pylibdmtx` или `reportlab` с GS1 DataMatrix

### Рекомендуемый размер
По инструкции WB: DataMatrix **22×22 мм**, шрифт чёрный.

## 3. Текущая реализация в FBS

### Что есть
| Компонент | Реализация |
|-----------|------------|
| Хранение КИЗ | `Order.kiz_code` (255 символов) — сохраняется полный код |
| Обрезка при complete | `kiz_code[:31]` для WB API |
| WB: ввод КИЗ | `add_kiz_code` → `PUT /api/v3/orders/{orderId}/meta/sgtin` |
| Ozon | КИЗ только сохраняется в БД, в API не передаётся (ship без exemplar) |
| Дубль этикетка | `/kiz-label` — DataMatrix (полный КИЗ) + 31 символ, PDF |
| Выгрузка | `/kiz-export` — xlsx/txt с КИЗ, posting_number, article, product_name |

### Что не так
1. ~~**Дубль этикетка** — QR вместо DataMatrix~~ — исправлено (DataMatrix + 31 символ)
2. **Выгрузка для WB** — нет формата «номер задания + номер стикера + 31 символ» для массовой загрузки
3. **Ozon** — КИЗ сохраняется, но при ship не передаётся (exemplar_info с is_gtd_absent)

## 4. Что нужно поправить

### 4.1. Дубль этикетки КИЗ (приоритет 1)
**Задача:** DataMatrix + 31 символ снизу.

- Заменить QR на **DataMatrix** в `/kiz-label`
- Кодировать в DataMatrix **полный** КИЗ (все символы, как пришли при сканировании)
- Под DataMatrix печатать **31 символ** текстом
- Использовать GS1 DataMatrix с FNC1 и разделителями GS (см. [markirovka.ru](https://markirovka.ru/community/best-practics/kak-sobiraetsya-kod-datamatrix))
- Размер: 22×22 мм (или пропорционально)

**Библиотеки:** `pylibdmtx`, `reportlab` (DataMatrixWidget), или `python-barcode` с DataMatrix.

### 4.2. Выгрузка для массовой загрузки WB (приоритет 2)
**Задача:** Шаблон с колонками: номер задания, номер стикера, 31 символ КИЗ.

- Добавить endpoint или формат выгрузки: **номер заказа (external_id)**, **номер стикера** (sticker_id из WB), **КИЗ 31 символ**
- WB Partners: «Скачать шаблон» → «Загрузить массово» — нужен точный формат колонок WB
- Уточнить в личном кабинете WB или документации формат шаблона для «Загрузить массово»

**Текущая выгрузка** (`/kiz-export`): КИЗ, posting_number, article, product_name, completed_at, marketplace.  
**Нужно добавить:** sticker_id (номер стикера) — его можно получить из WB API при синхронизации заказов.

### 4.3. Сохранение КИЗ (WB и Ozon)
- **WB:** уже передаётся через `add_kiz_code` при complete. Сохранять в БД — уже есть.
- **Ozon:** только сохранять в БД, в API не передавать — уже так.
- Рекомендация: хранить **полный** КИЗ в БД (сейчас `kiz_code` 255 символов — достаточно).

### 4.4. Режим «таблица вместо ввода в интерфейс»
**Задача:** Пользователь собирает заказы, сканирует КИЗ, в конце выгружает файл и загружает его в WB вручную.

- Реализовать выгрузку в формате WB (номер задания, номер стикера, 31 символ)
- Отключить или сделать опциональным вызов `add_kiz_code` при complete — чтобы КИЗ только сохранялись, а загрузка в WB шла через файл

## 5. Состав КИЗ (символы)

| Символы | Значение |
|---------|----------|
| `01` | Идентификатор применения (AI) |
| 14 цифр | GTIN |
| `21` | AI серийного номера |
| 13 символов | Серийный номер |
| `91` | AI ключа проверки |
| 4 символа | Ключ проверки |
| `92` | AI кода проверки |
| 4–88 символов | Криптохвост (для одежды 88) |

Символы `>`, `&`, `=`, `+`, `/` — часть криптохвоста или разделители. Их не нужно «расшифровывать» — они генерируются ЦРПТ и должны сохраняться как есть при передаче в DataMatrix.
