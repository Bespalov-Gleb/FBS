# План исправлений: размеры и фото товаров

## 1. Размеры товаров

### Проблема
Неправильно подгружаются размеры — отображается не тот размер, который заказан (например, «Размер на модели: 50-52» вместо фактического размера заказа «46»).

### Причина
- **Ozon:** `get_product_sizes` возвращал атрибут «Размер» — только цифры (44). Нужен «Размер продавца» — формат 46-48 / M с буквами. Реальный размер заказа может быть в `dimensions.size_name` в продукте posting.
- **WB:** размер берётся из Content API по `chrt_id` — при несовпадении может подставляться первый размер из списка.

### Исправления (сделано)

1. **Ozon** (`order_sync_service.py`): при установке `p["size"]` сначала проверяем `dimensions.size_name` / `dimensions.size` из продукта posting. Только если их нет — используем `get_product_sizes`.

2. **Ozon** (`ozon.py`): в `get_product_sizes` приоритет атрибута **«Размер продавца»** (формат 46-48 / M) над «Размер» (только цифры). Ищем сначала «размер продавца», затем «размер».

3. **WB:** логика `_extract_size_from_card` уже использует `chrt_id` для выбора размера по конкретному варианту. Убедитесь, что `chrt_id` передаётся в metadata при синхронизации заказов WB.

### WB размер (дополнение)
- **`wildberries.py`**: в `_extract_size_from_card` — нормализация `chrt_id` (int/str), fallback на первый размер при несовпадении.

## 2. Фото товаров

### Ozon (обновлено 2026-03)

**Проблема:** Не для всех товаров подтягиваются фото.

**Причины:**
- `product_id` и `sku` в Ozon — разные сущности; использовали sku как product_id.
- API product/info/list принимает один тип идентификатора за запрос: `offer_id` или `sku`.

**Исправления:**
1. Сначала запрос по `offer_id`, затем fallback по `sku` с маппингом `sku_to_article`.
2. Выбор фото: приоритет `primary_image` (главное фото Ozon), fallback `images[0]`.
3. Вынесена `_extract_product_image_url()` для единообразия.

### WB

**Проблема:** Второе фото (штрихкод) вместо основного.

**Исправления (сделано):**
- Content API: при нескольких фото берём `photos[1]` (основное), иначе `photos[0]`.
- WB CDN: пока без изменений.
