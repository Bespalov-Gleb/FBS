# Проверка: поддержка обрезки на уровне PDF и рендер по области

## 1. CropBox (обрезка страницы перед рендером)

### pypdf
- **Поддерживается.** Можно менять `page.mediabox` и `page.cropbox` (например, через `RectangleObject((left, bottom, right, top))`).
- Контент не удаляется, меняется только видимая область.
- **Ограничение:** pypdf **не умеет** считать bbox контента страницы. Координаты для CropBox нужно получать иначе (см. ниже).

### pdf2image
- **Поддерживается.** Параметр `use_cropbox=True` в `convert_from_bytes()` — рендер идёт по CropBox, а не по MediaBox.
- **Цепочка:** изменить CropBox в pypdf → сохранить PDF в bytes → `convert_from_bytes(pdf_bytes, use_cropbox=True)` → получим картинку уже без лишних полей.

### Как получить bbox без PyMuPDF
- Один раз отрендерить страницу (pdf2image), в пикселях найти bbox контента (порог «не белый»).
- Пересчитать пиксели в пункты PDF: `x_pt = x_px * 72 / dpi`, для оси Y учесть, что в PDF Y идёт снизу вверх: `y_pt_bottom = (height_px - y_max_px) * 72 / dpi`, `y_pt_top = (height_px - y_min_px) * 72 / dpi`.
- Размер страницы в пунктах взять из `page.mediabox` (pypdf).

## 2. Рендер только области страницы (clip)

### pdf2image
- **Не поддерживается.** Нет параметра «область/rect» — рендерится вся страница. Обрезка только постфактум по картинке (PIL).

### PyMuPDF (fitz)
- **Поддерживается.** `page.get_pixmap(clip=rect)` рендерит только прямоугольник `rect`.
- Есть API для bbox контента: `get_text("blocks")` (блоки с координатами), `get_drawings()` (векторная графика с `rect`).
- **В проекте PyMuPDF нет** (в `pyproject.toml` только pypdf и pdf2image). Вариант 2 потребовал бы добавить зависимость `pymupdf`.

## Итог

| Подход | pypdf | pdf2image | Без новых зависимостей |
|--------|--------|-----------|------------------------|
| 1. CropBox + повторный рендер | да (задать CropBox) | да (use_cropbox=True) | да (bbox из первого рендера в пикселях → пересчёт в пункты) |
| 2. Рендер по clip | — | нет | нет (нужен PyMuPDF) |

Реализуем **вариант 1**: один раз рендерим → считаем bbox в пикселях → переводим в пункты → выставляем CropBox в pypdf → сохраняем PDF → рендерим с `use_cropbox=True` и дальше текущий пайплайн (поворот, размещение на 58×40).
