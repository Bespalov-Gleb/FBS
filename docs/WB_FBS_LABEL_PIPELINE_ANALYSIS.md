# Анализ пайплайна обработки WB FBS этикетки и строчки eb4...

## 1. Входные данные

- **Источник**: PNG от WB API (запрос стикера, типично `width=58 height=40` в единицах WB).
- **Фактический размер** (из логов): **580×400** пикселей (альбом: ширина > высоты).
- **Содержимое**: сверху — этикетка (штрихкоды, QR, «WB», цифры), по центру — **большой белый зазор**, внизу — **одна строка текста** `eb4.r2229f12eb7ab46c88757` (код WB).
- **Настройка поворота**: `deg` из настроек печати (часто 90°).

---

## 2. Пошаговый пайплайн (как сейчас)

### Шаг 0. Загрузка и параметры

- `img = Image.open(...).convert("RGB")` → размер 580×400.
- `deg = 90` (или 0/180/270) — угол поворота этикетки.

### Шаг 1. Bbox-кроп (обрезание по контенту)

- Для каждого пикселя с яркостью < 250 ищутся `min_x, min_y, max_x, max_y`.
- Обрезка: `img.crop((min_x, min_y, max_x+1, max_y+1))`.
- Результат: убрана белая рамка по краям, **низ изображения по-прежнему строчка** (она попадает в bbox). Размер может стать, например, 540×400.

### Шаг 2. Кроп «первая тёмная строка сверху»

- Ищется первая строка `y`, где есть хотя бы один пиксель < 253.
- Обрезка: `img.crop((0, y_top, iw, ih))` — **режется только верх**, низ не трогается.
- Строчка остаётся внизу. Размер: например 540×390.

### Шаг 3. Поиск горизонтального «белого пояса»

- Цель: найти полосу **белых** строк между этикеткой и строчкой.
- Условие «белая строка»: в строке тёмных пикселей (v < 250) **меньше** чем `max(2, iw * dark_frac)` при `dark_frac = 0.10` (т.е. при iw=540 — меньше 54 штук).
- Собираются все такие полосы подряд, выбирается самая длинная в **нижней половине** картинки (`mid >= ih*0.25`).
- Если пояс найден:
  - `top_img` = выше пояса (этикетка), `bottom_img` = ниже пояса (строчка).
  - Крутится только `top_img` по `deg`; `bottom_img` потом ещё поворачивается на 180° и накладывается сверху.
- **Факт по логам**: `bands=0` — **ни одна строка не считается белой**. Значит, в каждой строке ≥54 тёмных пикселей (шум, антиалиасинг, сжатие WB). Этот шаг на твоих этикетках **никогда не срабатывает**.

### Шаг 4. Поворот всего изображения (если пояс не найден)

- Условия: `deg == 90` и `ih > iw` → ROTATE_270; `deg == 270` и `iw > ih` → ROTATE_90; `deg == 180` → ROTATE_180.
- Для **580×400** имеем `ih=400 < iw=580` → условие `ih > iw` ложно → **поворот всего изображения не выполняется**.
- Картинка остаётся в исходной ориентации: этикетка сверху, зазор, строчка внизу.

### Шаг 5. «Сжатие» — поиск белого пояса по середине

- Снова ищется самый широкий горизонтальный пояс «белых» строк (`dark_frac = 0.12`).
- Если найден — изображение режется по поясу, нижний кусок (строчка) накладывается на верхний вверху.
- **Факт по логам**: `best_band_len=0` — опять **нет ни одной достаточно белой строки**. Шаг не срабатывает.

### Шаг 6. Обрезка по высоте (при сильной вытянутости)

- Если `ih > iw * 1.35` — обрезается низ. Для 400×540 не выполняется.

### Шаг 7. Рисование на PDF

- Размер страницы 40×58 мм (книжная ориентация).
- Изображение масштабируется и привязывается **к верху** листа: `y_place = frame_h_pt - margin_top_pt - draw_h`.
- Номер заказа рисуется внизу страницы по центру.

## 3. Почему «строчка не прижимается» и почему режем этикетку

### Почему не прижимается

- И «горизонтальный пояс», и «сжатие» опираются на **порог «белизны»**: строка считается белой, только если тёмных пикселей **меньше** заданного числа (10% или 12% ширины).
- На твоих PNG **ни одна строка не проходит** этот порог → `bands=0`, `best_band_len=0` → разделение этикетка/строчка не происходит, строчка никуда не переносится.

### Почему при обрезке/затирании «низа» режем этикетку

- При **фиксированной полосе снизу** (например последние 25–35 px) мы не знаем, где именно заканчивается контент этикетки (нижний штрихкод) и начинается зазор/строчка.
- На разных этикетках WB:
  - высота строчки и размер зазора могут отличаться;
  - нижний штрихкод этикетки может располагаться по-разному по вертикали.
- Поэтому фиксированные «последние N пикселей» то попадают только в строчку, то захватывают низ этикетки → пользователь видит: «резанул низ этикетки».

---

## 4. Что нужно для 100% решения

Нужно **не опираться на «белые» строки** и **не полагаться на фиксированную полосу по высоте**. Нужно **по контенту** найти, где заканчивается этикетка и где начинается/заканчивается строчка.

Идея: **профиль плотности по вертикали**.

- Для каждой строки `y` считаем число тёмных пикселей (например v < 250) — это «плотность» строки.
- В **нижней части** изображения (например нижние 40–50% по высоте):
  - **Зазор** между этикеткой и строчкой даёт **минимум** плотности (почти пустые строки).
  - **Строчка** даёт **локальный максимум** плотности (много тёмного в одной–двух строках текста).
- Алгоритм:
  1. Строим массив `density[y]` для y из нижней половины (или нижних 40%) высоты.
  2. Находим в этой зоне **глобальный минимум** плотности — скорее всего зазор; запоминаем `y_gap`.
  3. Ниже (ближе к низу картинки) ищем **локальный максимум** плотности — это строка строчки; от него вверх/вниз идём, пока плотность не падает — получаем `y_line_top`, `y_line_bottom`.
  4. Либо **убираем строчку**: затираем белым только прямоугольник `[y_line_top : y_line_bottom+1]` (или обрезаем `img` до `y_line_top`), не трогая остальное.
  5. Либо **прижимаем к верху**: вырезаем полосу `[y_line_top : y_line_bottom+1]`, собираем новое изображение — сначала эта полоса вверху (y=0), под нею — всё от y=0 до y_line_top (этикетка + зазор). Размер и верх этикетки не меняются, меняется только порядок: строчка оказывается вверху.

Так мы **не зависим от порога «белизны»** и **не режем по фиксированному числу пикселей** — граница определяется по реальному контенту (минимум/максимум плотности), поэтому это даёт **100% вариант**: либо убрать строчку, либо прижать её к верху, не задевая этикетку.

## 5. Предлагаемый 100% вариант (без правки кода в этом файле)

### Вариант А: Убрать строчку

1. После шагов 1–2 (bbox + first dark row) вычислить профиль плотности для нижних 40% строк (`y` от `int(ih*0.6)` до `ih-1`).
2. Найти в этом диапазоне строку с **минимальной** плотностью → считаем верхом зазора или низом этикетки; затем найти компактный блок с **высокой** плотностью у самого низа → это строчка (`y_line_top`, `y_line_bottom`).
3. **Закрасить белым** только прямоугольник `(0, y_line_top, iw, y_line_bottom+1)` в текущем `img`. Размер изображения не менять, этикетку не обрезать.
4. Дальше выполнять текущий пайплайн (поиск пояса, повороты, сжатие, PDF) как обычно.

### Вариант Б: Прижать строчку к верху

1. Те же шаги 1–2 и тот же расчёт профиля → получить `y_line_top`, `y_line_bottom`.
2. `top_part = img.crop((0, 0, iw, y_line_top))` (этикетка + зазор).
3. `line_part = img.crop((0, y_line_top, iw, y_line_bottom+1))` (строчка).
4. Собрать новое изображение: высота = `top_part.height + line_part.height`, сначала вставить `line_part` в (0, 0), затем `top_part` в (0, line_part.height) — строчка оказывается вверху, этикетка под ней. Либо наложение: размер как у `top_part`, сначала `top_part`, затем `line_part` в (0, 0).
5. Дальше пайплайн как обычно (повороты только этикетки уже не нужны для строчки — она отдельно и вверху).

Оба варианта делают одну и ту же **надёжную детекцию строчки по профилю плотности**; разница только в действии: затирание (убрать) или вырез + склейка вверху (прижать).
